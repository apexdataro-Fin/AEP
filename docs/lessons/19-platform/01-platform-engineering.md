# هندسة المنصات

> **"بدلاً من أن يبني كل فريق بنيته التحتية من الصفر — ابنِ منصة داخلية تخدم الجميع."**

## ما هي منصة المطورين الداخلية (IDP)؟

طبقة تجريد فوق الأدوات والبنية التحتية. توفر:

| المكون           | مثال                     | يوفر                         |
| ---------------- | ------------------------ | ---------------------------- |
| **كتالوج خدمات** | Backstage                | اكتشف كل الخدمات والـ APIs   |
| **سقالات**       | Backstage Templates      | أنشئ خدمة جديدة بضغطة زر     |
| **CI/CD موحد**   | GitHub Actions Templates | كل خدمة تُنشر بنفس الطريقة   |
| **مراقبة موحدة** | Grafana Dashboards       | كل خدمة تُراقب بنفس المقاييس |
| **بوابة واحدة**  | Backstage                | مدخل واحد لكل شيء            |

## قبل وبعد المنصة

| الجانب              | قبل المنصة      | بعد المنصة      |
| ------------------- | --------------- | --------------- |
| إنشاء خدمة جديدة    | ٣ أسابيع        | ٣ ساعات         |
| طرق النشر           | ١٢ طريقة مختلفة | مسار ذهبي واحد  |
| معرفة ما في الإنتاج | لا أحد يعرف     | كتالوج كامل     |
| وقت المهندس         | ٤٠٪ في العمليات | ١٠٪ في العمليات |

## المسار الذهبي — Golden Path

```yaml
# قالب خدمة — كل مطور يستخدمه
name: "قالب خدمة Backend"
steps:
  - name: "اختر لغة"
    type: select
    options: [Go, Python, TypeScript]
  - name: "اختر قاعدة بيانات"
    type: select
    options: [PostgreSQL, MongoDB, None]
  - name: "أنشئ"
    action: scaffold
# النتيجة: Repo جديد مع CI/CD، Dockerfile، مراقبة — كل شيء جاهز
```

## من أين تبدأ؟

1. **اسأل المطورين:** ما أكثر شيء يهدر وقتك؟
2. **ابنِ MVP:** حل المشكلة الأكبر أولاً
3. **لا تفرض:** أظهر القيمة — "فريق X نشر في ١٠ دقائق"
4. **كرر:** أضف خدمات جديدة بناءً على الطلب
