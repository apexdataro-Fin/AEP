# أساسيات الحوسبة السحابية

> **"السحابة ليست 'حواسيب شخص آخر'. إنها نموذج جديد كلياً للتفكير في البنية التحتية."**

## لماذا السحابة؟

| قبل السحابة                     | بعد السحابة                       |
| ------------------------------- | --------------------------------- |
| شراء خوادم — يصلون بعد ٤ أسابيع | `az vm create` — ٩٠ ثانية         |
| توقع الحمل المستقبلي            | Auto-scaling — تابع للحظة         |
| دفع ثمن الخوادم حتى لو معطلة    | Pay-per-use — ادفع فقط لما تستخدم |
| ٣ مهندسين لإدارة العتاد         | ٠ مهندسين لإدارة العتاد           |
| مركز بيانات واحد = نقطة فشل     | عشرات المراكز حول العالم          |

## نماذج الخدمة — IaaS vs PaaS vs SaaS

| النموذج         | أنت تدير                   | السحابة تدير                      | مثال              | تكلفة الجهد |
| --------------- | -------------------------- | --------------------------------- | ----------------- | ----------- |
| **On-Premises** | كل شيء                     | لا شيء                            | خوادمك في مكتبك   | $$$$        |
| **IaaS**        | OS، middleware، apps، data | Hardware، virtualization، network | Azure VM          | $$$         |
| **PaaS**        | Apps، data                 | OS، runtime، middleware           | Azure App Service | $$          |
| **SaaS**        | لا شيء                     | كل شيء                            | Microsoft 365     | $           |

### تشبيه البيتزا 🍕

- **On-Premises:** تصنع البيتزا من الصفر في بيتك — تشتري الدقيق، تعجن، تخبز
- **IaaS:** تشتري بيتزا مجمدة — تخبزها في فرنك
- **PaaS:** تطلب بيتزا — توصل لباب بيتك، تأكل
- **SaaS:** تذهب للمطعم — كل شيء جاهز، تدفع وتستمتع

## نموذج المسؤولية المشتركة

```
🟦 Microsoft مسؤولة عن:          🟧 أنت مسؤول عن:
─────────────────────────       ────────────────────
• أمن مراكز البيانات            • بياناتك وتشفيرها
• العتاد المادي                  • نقاط النهاية والتطبيقات
• الشبكة المادية                 • الحسابات وإدارة الهوية
• المضيفين والمحاكيات            • تكوينات الأمان
                                 • الامتثال لسياساتك
```

**الدرس المهم:** Azure يؤمن البنية التحتية. لكن بياناتك — مسؤوليتك أنت.

## المناطق ومناطق التوفر

```
Azure Region: West Europe
├── AZ1: مركز بيانات أمستردام — طاقة مستقلة
├── AZ2: مركز بيانات روتردام — تبريد مستقل
└── AZ3: مركز بيانات لاهاي — شبكة مستقلة
```

| المفهوم           | ماذا يعني                       | مثال                       |
| ----------------- | ------------------------------- | -------------------------- |
| **Region**        | منطقة جغرافية                   | West Europe، East US       |
| **AZ**            | مراكز بيانات مستقلة داخل Region | 3 AZ in West Europe        |
| **Fault Domain**  | رف Rack يشارك مصدر طاقة         | Rack #7 في AZ1             |
| **Update Domain** | مجموعة تُحدّث معاً              | Update Domain 0            |
| **Region Pair**   | منطقتان مرتبطتان للتعافي        | West Europe ↔ North Europe |

| **الإجمالي**     |              | **~$٧٧٤** |

### استراتيجيات التوفير

```bash
# ١. Reserved Instances — وفر حتى ٧٢٪
# بدل pay-as-you-go، ادفع مقدماً سنة أو ٣ سنوات

# ٢. Auto-shutdown — وفر ٦٥٪ من dev/test
az vm auto-shutdown -g dev-rg -n dev-vm --time 2000

# ٣. الحجم المناسب
# Standard_D4s_v3 (4 cores, 16GB) → $١٥٠/شهر
# Standard_B2ms   (2 cores,  8GB) → $٦٠/شهر
# ← إذا الـ CPU average < 20%، اختر الأصغر

# ٤. احذف غير المستخدم
az disk list --query "[?managedBy==null].name"  # أقراص يتيمة
az network public-ip list --query "[?ipAddress==null].name"  # IPs غير مستخدمة
```

## سيناريو CloudNova: اختيار النموذج

> **الموقف:** CloudNova تبني منصة API جديدة. فريق ٤ مطورين. ميزانية محدودة.

### تحليل الخيارات:

| الخيار                 | الإيجابيات                   | السلبيات                       | التكلفة         |
| ---------------------- | ---------------------------- | ------------------------------ | --------------- |
| **IaaS (VMs)**         | تحكم كامل                    | إدارة OS، تحديثات، نسخ احتياطي | $٣٠٠ + وقت كثير |
| **PaaS (App Service)** | لا إدارة OS، تحديثات تلقائية | أقل مرونة في التكوين           | $٢٩٢            |
| **AKS (Kubernetes)**   | مرونة عالية، مستقبلي         | يحتاج خبرة K8s، Overhead عالي  | $٤٠٠ +          |

**التوصية:** PaaS الآن. **اختر البساطة.** إذا نجح المنتج وكبر الفريق — انتقل لـ Kubernetes.

**القاعدة الذهبية:** ابدأ بأقل تعقيد ممكن. لا تبالغ في الهندسة قبل أن تحتاج.
