# المسار الوظيفي في السحابة

> "مهندس السحابة ليس مجرد وظيفة. إنه رحلة مستمرة من التعلم."

## 🎯 أهداف التعلم

- فهم خريطة المسار الوظيفي لمهندس السحابة
- معرفة الشهادات المطلوبة في كل مرحلة
- بناء خطة تطوير مهني
- فهم سوق العمل والرواتب
- تجنب الأخطاء الشائعة في المسار الوظيفي

## 📖 الطبقة الأساسية: سلمCloud Engineer الوظيفي

### المراحل الوظيفية الخمس

```
المرحلة 1: Junior Cloud Engineer (0-2 سنوات)
├── الراتب: $65K - $90K
├── المهام: Tickets، مراقبة، نشر بسيط
├── الشهادات: AZ-900
└── المهارات: Linux، أساسيات Azure، Git

المرحلة 2: Cloud Engineer (2-4 سنوات)
├── الراتب: $90K - $130K
├── المهام: تصميم الحلول، أتمتة، CI/CD
├── الشهادات: AZ-104
└── المهارات: Terraform، Docker، Kubernetes

المرحلة 3: Senior Cloud Engineer (4-7 سنوات)
├── الراتب: $130K - $170K
├── المهام: Architecture، Mentoring، Incidents
├── الشهادات: AZ-400 (DevOps)، AZ-305 (Architect)
└── المهارات: Observability، FinOps، Security

المرحلة 4: Cloud Architect (7-10 سنوات)
├── الراتب: $170K - $220K
├── المهام: تصميم المؤسسة، حوكمة، استراتيجية
├── الشهادات: AZ-305 (الخبرة أهم من الشهادة)
└── المهارات: Multi-cloud، Enterprise patterns، Leadership

المرحلة 5: Principal/Director (10+ سنوات)
├── الراتب: $200K - $300K+
├── المهام: رؤية تقنية، فرق متعددة، قرارات استراتيجية
└── المهارات: Business acumen، People management
```

## 🏗️ الطبقة الإنتاجية: CloudNova Career Mode

### رحلتك في CloudNova

```
أنت: Junior Cloud Engineer في CloudNova

الفصل 1: Onboarding (أسبوع 1-2)
├── المهمة: إعداد بيئة التطوير
├── التعلم: Azure Portal، Azure CLI
└── المهارة: أساسيات Azure

الفصل 2: أول مهمة (أسبوع 3-4)
├── المهمة: نشر تطبيق بسيط على App Service
├── التعلم: CI/CD، GitHub Actions
└── المهارة: أتمتة النشر

الفصل 3: حادثة إنتاجية (أسبوع 5-6)
├── المهمة: تشخيص ارتفاع CPU في الإنتاج
├── التعلم: Monitoring، استكشاف الأخطاء
└── المهارة: حل المشكلات

الفصل 4: ترقية (شهر 6)
├── الترقية: Cloud Engineer
├── المسؤوليات: تصميم الحلول، أتمتة
└── الشهادة: AZ-104

الفصل 5: قيادة مشروع (سنة 2)
├── الترقية: Senior Cloud Engineer
├── المشروع: هجرة إلى Kubernetes
└── الفريق: 3 مهندسين
```

## 🏥 سيناريو CloudNova: مقابلة العمل

```
📋 المشهد: مقابلتك الأولى لـ Cloud Engineer

السؤال 1: "احكِ لي عن مشروع صعب عملت عليه"

الإجابة المثال:
"في CloudNova، واجهتنا مشكلة بطء النشر.
كنت أنشر يدوياً وكان يأخذ 4 ساعات.
صممت CI/CD Pipeline مع GitHub Actions + Argo CD.
الآن النشر يأخذ 10 دقائق.
تعلمت: الاستثمار في الأتمتة يعود 100x."

الهيكل:
├── المشكلة (سياق)
├── الحل (ماذا فعلت أنت)
├── النتيجة (أرقام!)
└── الدرس (ماذا تعلمت)

السؤال 2: "كيف تتعامل مع Incident في الإنتاج؟"

"أولاً، لا panic. أفتح الـ runbook.
1. Diagnose: أتحقق من Dashboards و Logs
2. Mitigate: أوقف النزيف أولاً (rollback إذا لزم)
3. Resolve: أصلح السبب الجذري
4. Learn: Post-mortem لمنع التكرار"
```

## 📝 بطاقات تعليمية

- **AZ-104**: شهادة Azure Administrator — بوابتك لسوق العمل
- **CKA**: شهادة Kubernetes — الأكثر طلباً بعد AZ-104
- **FinOps**: مهارة إدارة التكاليف — تميزك عن غيرك
- **System Design**: مهارة تصميم الأنظمة — للترقية لـ Architect
- **GitHub Portfolio**: دليل عملك — أهم من الشهادات أحياناً

## 🎤 أسئلة المقابلة

1. **"أين ترى نفسك بعد 3 سنوات؟"**
   - "أرى نفسي Cloud Architect أقود تصميم حلول معقدة.
     أخطط للحصول على AZ-305 وقيادة مشاريع هجرة."
   - لا تقل "لا أعرف" أو "في أي مكان"

2. **"ما أكبر خطأ ارتكبته؟"**
   - "حذفت production database بالخطأ.
     لحسن الحظ كان معنا backups.
     تعلمت: double-check قبل التنفيذ،
     وأنشأت safeguards تلقائية."

3. **"لماذا تريد العمل معنا؟"**
   - ابحث عن الشركة قبل المقابلة
   - اربط مهاراتك باحتياجاتهم
   - كن محدداً (ليس "لأنكم شركة كبيرة")
