# تحسين السيرة الذاتية و LinkedIn

> "سيرتك الذاتية لديها 6 ثوانٍ لتثير الإعجاب. اجعل كل كلمة تحسب."

## 🎯 أهداف التعلم

- كتابة سيرة ذاتية تجتاز ATS
- تحسين LinkedIn Profile
- Keywords المناسبة لـ Cloud Engineer
- أمثلة عملية

## ⏱️ الوقت المقدر: 30 دقيقة | المستوى: Junior

---

## 🏗️ قواعد السيرة الذاتية

### 1. ATS Optimization

ATS (Applicant Tracking System) يبحث عن keywords:

```
✅ صحيح: "نشرت Kubernetes cluster على Azure AKS لـ 50 خدمة"
❌ خاطئ: "عملت على مشاريع cloud"

✅ صحيح: "خفضت فاتورة Azure بنسبة 40% عبر right-sizing و Reserved Instances"
❌ خاطئ: "وفرت فلوس للشركة"
```

### 2. هيكل bullet points

```
[فعل] + [ماذا] + [كيف] + [نتيجة/رقم]

مثال:
"أتمتة [فعل] نشر البنية التحتية [ماذا] باستخدام Terraform و CI/CD [كيف]
مما قلل وقت النشر من 3 أيام إلى 15 دقيقة [نتيجة]"
```

### LinkedIn Optimization

| العنصر       | أفضل ممارسة                                         |
| ------------ | --------------------------------------------------- |
| **الصورة**   | احترافية، خلفية نظيفة                               |
| **العنوان**  | "Cloud & DevOps Engineer                            | Azure | Kubernetes | Terraform" |
| **About**    | قصة قصيرة: من أنت + ماذا تفعل + ماذا تريد           |
| **المهارات** | Azure, Kubernetes, Terraform, Docker, CI/CD, Python |
| **التوصيات** | اطلب من زملاء ومديرين سابقين                        |

1. راجع سيرتك الذاتية الحالية واحذف أي bullet point بدون رقم
2. أضف 3 keywords من وصف وظيفة أحلامك
3. حسن LinkedIn headline

## 🏛️ CloudNova: سيرة ذاتية فتحت باب المقابلة

**لمياء** قدمت على وظيفة Cloud Engineer في CloudNova. 300 متقدم. 10 فقط وصلوا للمقابلة.

**لماذا نجحت سيرتها؟**

ATS scan وجد:

- ✅ "Azure" — 7 مرات
- ✅ "Kubernetes" — 5 مرات
- ✅ "Terraform" — 4 مرات
- ✅ "CI/CD" — 3 مرات
- ✅ "Python" — 4 مرات

**نموذج bullet point القاتل:**

```
❌ "عملت على مشاريع cloud"
✅ "صممت ونشرت AKS cluster لـ 30 microservice، مما قلل deployment time من 45 دقيقة إلى 8 دقائق (توفير 80%)"
```

**الفرق:**

- ❌ غامض، لا measurable impact
- ✅ محدد (AKS, 30 service)، رقمي (80%)، يوضح القيمة

### صيغة bullet point السحرية

```
[فعل قوي] + [ماذا بالضبط] + [كيف/بأي تقنية] + [نتيجة برقم]

أمثلة:
1. "أتمتة نشر البنية التحتية لـ 50 تطبيقاً باستخدام Terraform و GitHub Actions،
   مما قلل وقت النشر من 3 أيام إلى 15 دقيقة"

2. "خفضت فاتورة Azure الشهرية بنسبة 40% عبر right-sizing و Reserved Instances،
   مما وفر $180,000 سنوياً"

3. "قاد ترحيل 10 تطبيقات من VMs إلى AKS دون downtime،
   محسناً scalability بنسبة 300%"
```

## 🎨 أخطاء قاتلة في السيرة الذاتية

| الخطأ          | لماذا هو قاتل           | الإصلاح                                                       |
| -------------- | ----------------------- | ------------------------------------------------------------- |
| **هدف عام**    | "أبحث عن فرصة في cloud" | هدف محدد: "Senior Cloud Engineer متخصص في Azure و Kubernetes" |
| **وصف وظيفي**  | نسخ job description     | أضف إنجازاتك أنت                                              |
| **بدون أرقام** | "حسنت الأداء"           | "حسنت latency بنسبة 60%"                                      |
| **طويلة جداً** | 3+ صفحات                | صفحة واحدة (لأقل من 10 سنوات خبرة)                            |
| **تنسيق معقد** | جداول، صور، أعمدة       | بسيط، ATS-friendly                                            |

### LinkedIn Optimization Checklist

- [ ] صورة احترافية (خلفية نظيفة، إضاءة جيدة)
- [ ] Headline: "Cloud & DevOps Engineer | Azure | Kubernetes | Terraform"
- [ ] About: 3-4 جمل — من أنت، ماذا تفعل، ماذا تريد
- [ ] Featured: أفضل مقال أو مشروع
- [ ] Experience: bullet points برقم لكل منصب
- [ ] Skills: 50 مهارة (top 3 pinned)
- [ ] Recommendations: 3+ من زملاء ومديرين
- [ ] URL مخصص: linkedin.com/in/yourname
- [ ] Open to Work: بحذر (قد يظهر يائساً)

## 🛠️ تدريبات عملية

### تمرين 1: حول bullet points ضعيفة إلى قوية

```markdown
حول هذه:
❌ "مسؤول عن Kubernetes"
❌ "عملت على Terraform"
❌ "ساعدت في تقليل التكاليف"
❌ "شاركت في مشروع ترحيل"
❌ "أدير فريق DevOps"

إلى:
✅ ... (أعد كتابتها بصيغة [فعل] + [ماذا] + [كيف] + [رقم])
```

### تمرين 2: اختبار ATS

```bash
# انسخ سيرتك الذاتية ووصف وظيفة أحلامك
# مرر الكلمتين في محلل ATS

# تحقق من:
# - كم keyword مشترك بين السيرة والوصف؟
# - هل المهارات المطلوبة موجودة؟
# - هل يوجد bullet points برقم؟
# الهدف: 80%+ match
```

### تحدي: سيرة ذاتية من صفحة واحدة

```markdown
تحدي: اختصر كل خبراتك في صفحة واحدة (A4)

- احذف أي bullet point بدون رقم
- احذف أي وظيفة قبل 10 سنوات
- احذف "References available upon request"
- اختصر التعليم في سطرين
- استخدم font size 10-11 لتوفير مساحة
```

## 📝 تقييم

### ✅ Knowledge Checks

1. ما صيغة bullet point المثالية؟
2. كم كلمة مفتاحية يجب أن تطابق وصف الوظيفة؟
3. لماذا صفحة واحدة أفضل من صفحتين؟
4. ما الفرق بين objective و summary؟
5. كيف تكتب Experience section لـ ATS؟

### 🧠 Quiz

**س1:** أفضل bullet point:

- أ) "عملت على Kubernetes"
- ب) "نشرت 30 خدمة على AKS، محسناً scalability 300%" ✅
- ج) "مسؤول عن Kubernetes"
- د) "Kubernetes"

**س2:** ATS يبحث عن:

- أ) التصميم الجميل
- ب) الكلمات المفتاحية ✅
- ج) الصور
- د) الخطوط

**س3:** سيرة ذاتية مثالية:

- أ) 5 صفحات
- ب) صفحة واحدة مركزة ✅
- ج) 10 صفحات
- د) بدون نص

### 🗣️ Active Recall

1. اكتب bullet point مثالي عن آخر مشروع لك
2. صف LinkedIn Profile قوي لـ Cloud Engineer
3. كيف تعرف أن سيرتك جاهزة لـ ATS؟
4. ما أكبر خطأ في السير الذاتية العربية؟

### 🎓 Feynman Exercise

> اشرح ATS لغير تقني: "مثل غربال آلي. يقرأ 300 سيرة في دقائق ويبحث عن كلمات محددة. إذا سيرتك لا تحتوي الكلمات الصحيحة، لا يصل لبشر أبداً."

### 🃏 بطاقات تعلم

| السؤال                  | الإجابة                             |
| ----------------------- | ----------------------------------- |
| صيغة bullet point؟      | فعل + ماذا + كيف + رقم              |
| كم صفحة؟                | صفحة واحدة (لأقل من 10 سنوات)       |
| ما ATS؟                 | نظام تتبع المتقدمين — يفحص keywords |
| أفضل headline LinkedIn؟ | "Role                               | Tech1 | Tech2 | Tech3" |
| أهم عنصر في السيرة؟     | إنجازات برقم                        |
