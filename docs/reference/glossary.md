# Glossary

## A

**Auto Scaling**
: Automatically adjusting compute capacity based on demand. Implemented by services like AWS Auto Scaling Groups.

**Availability Zone (AZ)**
: Isolated locations within a cloud region. Each AZ has independent power, cooling, and networking.

## C

**CI/CD**
: Continuous Integration / Continuous Delivery. Automated pipeline for building, testing, and deploying software.

**Container**
: Lightweight, portable unit of software that packages code and dependencies together. Examples: Docker, containerd.

**Cloud Native**
: Approach to building and running applications that fully exploit cloud computing. Characterized by containers, microservices, and declarative APIs.

## D

**DevOps**
: Set of practices combining software development (Dev) and IT operations (Ops). Aims to shorten the development lifecycle.

## I

**IaC (Infrastructure as Code)**
: Managing infrastructure through machine-readable definition files. Tools: Terraform, Pulumi, CloudFormation.

**IAM (Identity and Access Management)**
: Framework for managing digital identities and their permissions to resources.

## K

**Kubernetes (K8s)**
: Open-source platform for automating deployment, scaling, and management of containerized applications.

## L

**Load Balancer**
: Device or service that distributes network traffic across multiple servers.

## M

**Microservices**
: Architectural style where an application is composed of small, independent services.

## O

**Observability**
: Ability to understand the internal state of a system from its external outputs. Pillars: logs, metrics, traces.

**Orchestration**
: Automated configuration, coordination, and management of computer systems and software.

## R

**Region**
: Geographic area containing multiple Availability Zones.

**Replication**
: Process of copying data across multiple systems for redundancy.

## S

**Serverless**
: Cloud computing model where the provider manages server allocation. Examples: AWS Lambda, Azure Functions.

**SLA (Service Level Agreement)**
: Commitment between a service provider and client about service availability and performance.

**SRE (Site Reliability Engineering)**
: Software engineering approach to IT operations. Uses automation and software to manage systems.

## T

**Terraform**
: Open-source IaC tool by HashiCorp. Uses HCL (HashiCorp Configuration Language).

## V

**Virtual Private Cloud (VPC)**
: Isolated virtual network within a public cloud.
