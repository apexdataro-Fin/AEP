## Simple Explanation

**The Azure SDK is like a remote control for every Azure service.**

Instead of clicking in the portal, you write Python code that says "Create a VM with these specifications." The SDK translates your Python into the REST API calls that Azure understands.

You could use the REST API directly (raw HTTP requests), but the SDK gives you Python objects, autocomplete, and error handling for free.

### Authentication: The First Step

```mermaid
sequenceDiagram
    participant Script as Python Script

    participant AAD as Azure AD

    Script->>DAC: "I need a token"
```

### SDK vs REST API

| Aspect | Azure SDK | REST API (requests) |
|--------|-----------|-------------------|
| **Setup** | `pip install azure-mgmt-compute` | `pip install requests` |
| **Auth** | `DefaultAzureCredential()` | Manual token management |
| **Pagination** | Automatic (`list()` yields all) | Manual `nextLink` handling |
| **Async ops** | `.result()` or LRO polling | Manual status checking |
| **Type safety** | Python objects with types | Raw JSON |
| **Coverage** | Most services | Every single Azure API |
| **Use when** | Standard operations | New APIs, unsupported services |

### Managing VMs with the SDK

<CodeBlock language="python" title="vm_manager.py">
{`"""Create, list, start, stop VMs programmatically."""
from azure.identity import DefaultAzureCredential
from azure.mgmt.compute import ComputeManagementClient
from azure.mgmt.network import NetworkManagementClient

credential = DefaultAzureCredential()
subscription_id = "your-subscription-id"

compute_client = ComputeManagementClient(credential, subscription_id)
network_client = NetworkManagementClient(credential, subscription_id)

# LIST: Get all VMs in a resource group
def list_vms(resource_group):
    vms = compute_client.virtual_machines.list(resource_group)
    for vm in vms:
        # SDK auto-handles pagination!
        power = compute_client.virtual_machines.instance_view(
            resource_group, vm.name
        ).statuses[1].display_status
        print(f"🖥️  {vm.name:<20} | {vm.hardware_profile.vm_size:<20} | {power}")

# CREATE: Deploy a VM (simplified — network must exist)
def create_vm(resource_group, vm_name, image="Ubuntu2204"):
    print(f"Creating {vm_name}...")
    
    # Long-running operation: SDK returns a poller
    poller = compute_client.virtual_machines.begin_create_or_update(
        resource_group,
        vm_name,
        {
            "location": "eastus",
            "hardware_profile": {"vm_size": "Standard_D2s_v3"},
            "storage_profile": {
                "image_reference": {
                    "publisher": "Canonical",
                    "offer": "0001-com-ubuntu-server-jammy",
                    "sku": "22_04-lts-gen2",
                    "version": "latest"
                }
            },
            "os_profile": {
                "computer_name": vm_name,
                "admin_username": "azureuser",
                "admin_password": os.environ["VM_ADMIN_PASSWORD"]  # NEVER hardcode!
            },
            "network_profile": {
                "network_interfaces": [{"id": f"/subscriptions/{subscription_id}/.../nic-{vm_name}"}]
            }
        }
    )
    
    # Wait for completion (or check poller.status() for progress)
    vm = poller.result()
    print(f"✅ VM {vm_name} ready at {vm.name}")

# STOP: Deallocate (save cost!)
def stop_vm(resource_group, vm_name):
    poller = compute_client.virtual_machines.begin_deallocate(resource_group, vm_name)
    poller.result()
    print(f"⏸️  {vm_name} deallocated — compute costs stopped")`}
</CodeBlock>

<Info>
**Long-Running Operations (LROs):** `begin_create_or_update()` and `begin_deallocate()` return pollers. Call `.result()` to block until complete, or `.status()` to check progress. For scripts that create 50 VMs, use `.wait()` with timeouts.
</Info>

## Production Explanation

### CloudNova: Bulk VM Operations

<ProductionNote title="Scenario: Cost Optimization">
CloudNova has 200 VMs across 3 subscriptions. The finance team wants to automatically deallocate all dev VMs at 8 PM and start them at 7 AM to save 50% on compute costs.
</ProductionNote>

<CodeBlock language="python" title="auto_shutdown.py">
{`"""Auto-shutdown dev VMs after 8 PM. Run via cron/Azure Automation."""
from azure.identity import DefaultAzureCredential
from azure.mgmt.compute import ComputeManagementClient
from azure.mgmt.resource import SubscriptionClient
from datetime import datetime

credential = DefaultAzureCredential()
sub_client = SubscriptionClient(credential)
current_hour = datetime.now().hour

# Skip if it's business hours
if 7 <= current_hour < 20:
    print("Business hours — skipping shutdown")
    exit(0)

stopped = 0
for sub in sub_client.subscriptions.list():
    compute = ComputeManagementClient(credential, sub.subscription_id)
    
    for vm in compute.virtual_machines.list_all():
        # Only touch dev VMs
        tags = vm.tags or {}
        if tags.get("environment") != "dev":
            continue
        
        # Skip already stopped
        instance_view = compute.virtual_machines.instance_view(
            vm.resource_group, vm.name
        )
        power_state = instance_view.statuses[1].display_status
        
        if power_state == "VM running":
            print(f"Stopping {vm.name}...")
            compute.virtual_machines.begin_deallocate(
                vm.resource_group, vm.name
            ).wait()
            stopped += 1

print(f"✅ Stopped {stopped} dev VMs")`}
</CodeBlock>

### Calling REST APIs Directly

<TerminalBlock>
{`# When the SDK doesn't support a new API, use REST directly
# Example: Delete all diagnostic settings (new API, limited SDK)

import requests
from azure.identity import DefaultAzureCredential

credential = DefaultAzureCredential()
token = credential.get_token("https://management.azure.com/.default").token

headers = {
    "Authorization": f"Bearer {token}",
    "Content-Type": "application/json"
}

# List all diagnostic settings for a resource
resource_id = "/subscriptions/.../resourceGroups/.../providers/Microsoft.Compute/virtualMachines/vm01"
url = f"https://management.azure.com{resource_id}/providers/Microsoft.Insights/diagnosticSettings?api-version=2021-05-01-preview"

response = requests.get(url, headers=headers)
response.raise_for_status()  # Raise exception on 4xx/5xx

settings = response.json()["value"]
print(f"Found {len(settings)} diagnostic settings")`}
</TerminalBlock>
