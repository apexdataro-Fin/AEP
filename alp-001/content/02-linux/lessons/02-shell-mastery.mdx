# Shell & Command Line Mastery

:::level simple

The command line looks scary — black screen, blinking cursor, no icons. But it's actually just texting with your computer. Instead of clicking buttons, you type what you want.

**Why it matters for cloud:** You can't click buttons on 50 servers at once. But you can run one command across all of them. That's why cloud engineers live in the terminal.

:::

:::level core

## The Big Three: stdin, stdout, stderr


Every Linux command has three standard streams:
- **stdin (0):** Where input comes from (keyboard or file or another command)
- **stdout (1):** Where normal output goes (terminal or file or another command)
- **stderr (2):** Where error messages go

:::

## Essential Commands

| Command | Purpose | CloudNova Use Case |
|---|---|---|
| `grep` | Search text | Find errors in logs |
| `find` | Find files | Locate config files |
| `awk` | Process columns | Extract IPs from logs |
| `sed` | Stream edit | Replace text in configs |
| `curl` | HTTP client | Test API endpoints |
| `jq` | JSON processor | Parse API responses |

### Pipes: The Unix Superpower

```bash
# Find all ERROR lines, extract the IP, count occurrences, show top 10
grep "ERROR" /var/log/app.log \
  | awk '{print $1}' \
  | sort \
  | uniq -c \
  | sort -rn \
  | head -10
```

### Redirection Quick Reference

```bash
command > file.txt   # stdout to file (overwrite)
command >> file.txt  # stdout to file (append)
command 2> error.log # stderr to file
command &> all.log   # both stdout and stderr to file
command < input.txt  # read stdin from file
```

### First Shell Script

```bash
#!/bin/bash
# check-service.sh — Check if a service is running

SERVICE=$1

if systemctl is-active --quiet "$SERVICE"; then
    echo "✅ $SERVICE is running"
    exit 0
else
    echo "❌ $SERVICE is NOT running"
    exit 1
fi
```

## Key Takeaways

- **stdin (0), stdout (1), stderr (2)** — the three I/O streams every command uses.
- **Pipes (`|`)** connect command output to command input — the Unix superpower.
- **Redirection (`>`, `>>`, `<`)** sends output to files or reads from files.
- **Shell scripts** are just commands in a file — start automating immediately.

## Check Your Understanding

1. **What happens when you run `command > output.txt 2>&1`?**
   - A) stdout goes to output.txt, stderr goes to terminal
   - B) Both stdout and stderr go to output.txt
   - C) Nothing — it's invalid syntax
   - D) Only stderr goes to output.txt

   <details><summary>Answer</summary>**B.** `2>&1` redirects stderr (file descriptor 2) to wherever stdout (file descriptor 1) is going — which in this case is `output.txt`.</details>
