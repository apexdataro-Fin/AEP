# Linux Security Hardening

:::level simple

**Security is not optional.** An unsecured Linux server exposed to the internet will be compromised within hours — sometimes minutes — by automated attacks.

Hardening means applying a set of configurations that reduce your attack surface. Think of it like locking doors and windows: you don't wait until after a burglary to install locks.

:::

:::level core

## SSH Hardening

```bash
# /etc/ssh/sshd_config
PermitRootLogin no               # Never allow root SSH
PasswordAuthentication no         # Keys only, no passwords
PubkeyAuthentication yes          # Allow SSH keys
MaxAuthTries 3                    # Limit brute force attempts
ClientAliveInterval 300           # Disconnect idle sessions after 5 min
AllowUsers david emma             # Only specific users can SSH
```

```bash
# After changing config, test and reload
sshd -t                           # Test config syntax
systemctl reload sshd             # Apply changes without dropping connections
```

## Firewall with UFW

```bash
# CloudNova Standard Firewall Rules
ufw default deny incoming         # Block everything by default
ufw default allow outgoing        # Allow all outbound

ufw allow 22/tcp                  # SSH
ufw allow 80/tcp                  # HTTP
ufw allow 443/tcp                 # HTTPS

ufw enable                        # Turn it on
ufw status verbose                # Verify
```

## Fail2ban: Block Attackers

```bash
apt install fail2ban

# /etc/fail2ban/jail.local
[DEFAULT]
bantime = 3600                    # Ban for 1 hour
findtime = 600                    # Window: 10 minutes
maxretry = 5                      # 5 failures = ban

[sshd]
enabled = true
```

## Automatic Security Updates

```bash
apt install unattended-upgrades
dpkg-reconfigure unattended-upgrades  # Choose "Yes"
```

---

<Example title="CloudNova Security Baseline Script">

```bash
#!/bin/bash
# cloudnova-harden.sh — Production hardening script
set -euo pipefail
echo "=== CloudNova Server Hardening ==="

# 1. SSH Hardening
sed -i 's/^#PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^#PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl reload sshd
echo "✅ SSH hardened"

# 2. Firewall
ufw --force enable
ufw default deny incoming
ufw allow 22/tcp
ufw allow 80,443/tcp
echo "✅ Firewall configured"

# 3. Auto updates
apt install -y unattended-upgrades
echo "✅ Auto updates enabled"

# 4. Fail2ban
apt install -y fail2ban
systemctl enable --now fail2ban
echo "✅ Fail2ban active"

echo "=== Hardening complete ==="
```

</Example>

## Key Takeaways

- **Disable root SSH login and password auth** — keys only.
- **Default deny firewall** — open only what you need.
- **fail2ban** blocks brute-force attacks automatically.
- **Automatic security updates** patch CVEs without manual intervention.
- Apply these to **every server, every time** — no exceptions.
