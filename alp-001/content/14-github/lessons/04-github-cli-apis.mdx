# GitHub CLI, APIs & Automation

<Definition>

The **GitHub CLI (`gh`)** brings pull requests, issues, releases, and workflows to your terminal. Combined with the REST and GraphQL APIs, you can automate every aspect of your development workflow.

</Definition>

---

## 🎯 Learning Objectives

- Master `gh` CLI for day-to-day operations
- Query and mutate GitHub data with REST and GraphQL
- Build automation scripts that integrate GitHub with cloud workflows

---

## 🔥 `gh` CLI — Terminal-First GitHub

<CodeBlock language="bash" title="Essential `gh` Commands">
# Authentication
gh auth login

# Pull Requests

gh pr create --title "feat: add cost tagging" --body "Closes #42"
gh pr list --state open
gh pr view 123
gh pr checkout 123
gh pr review --approve
gh pr merge --squash

# Issues

gh issue create --title "Bug: deployment timeout" --label "bug"
gh issue list --assignee @me
gh issue close 42

# Releases

gh release create v1.2.0 --title "v1.2.0" --notes "Added cost tagging"

# Workflows

gh run list
gh run watch
gh run view --log

# Repositories

gh repo clone apexdataro-Fin/AEP
gh repo fork
gh repo view --web

</CodeBlock>

## 🏗️ GitHub API — REST

<CodeBlock language="bash" title="REST API Examples">
# List open PRs
curl -H "Authorization: token $GH_TOKEN" \
  https://api.github.com/repos/apexdataro-Fin/AEP/pulls?state=open

# Create an issue

curl -X POST \
 -H "Authorization: token $GH_TOKEN" \
 -H "Content-Type: application/json" \
 -d '{"title":"Cost optimization report","body":"Run weekly cost analysis"}' \
 https://api.github.com/repos/apexdataro-Fin/AEP/issues

# Get workflow runs

curl -H "Authorization: token $GH_TOKEN" \
 https://api.github.com/repos/apexdataro-Fin/AEP/actions/runs

</CodeBlock>

## 🏭 Production Pattern: Release Automation

<CodeBlock language="bash" title="Automated Release Script">
#!/bin/bash
# scripts/release.sh — Automated release creation

VERSION=$1
if [ -z "$VERSION" ]; then
echo "Usage: ./release.sh v1.2.3"
exit 1
fi

# 1. Verify main is clean and up to date

git checkout main
git pull --rebase

# 2. Create tag

git tag -a "$VERSION" -m "Release $VERSION"

# 3. Push tag

git push origin "$VERSION"

# 4. Generate changelog and create GitHub release

gh release create "$VERSION" \
  --title "Release $VERSION" \
  --notes "$(git log --oneline $(git describe --tags --abbrev=0 @^)..@)" \
 --draft

echo "✅ Draft release $VERSION created. Review and publish."

</CodeBlock>
